# Convex Backend — Agent Guidelines

## TS2589 Prevention (Type Instantiation Depth)

TypeScript error TS2589 ("Type instantiation is excessively deep and possibly infinite") occurs when the generated `convex/_generated/api.d.ts` type tree exceeds TypeScript's 50-level recursion limit. This has been fixed and must not regress.

### Rules

1. **Never name a file the same as a sibling directory.**
   - `convex/auth.ts` alongside `convex/auth/` triggers TS2589 because Convex generates conflicting type paths.
   - The fix was renaming `auth.ts` to `authSetup.ts`. If you need a new top-level module, check that no directory shares the name.
   - This applies to any `convex/<name>.ts` where `convex/<name>/` also exists.

2. **Keep `legacyComponentApi: false` in `convex.json`.**
   - This generates import-based references for component types instead of inlining them, reducing type tree depth.
   - Never add `staticApi: true` — it eliminates TS2589 but introduces implicit `any` return types on a large percentage of functions, destroying type safety.

3. **Prefer `internal.*` over `api.*` in actions calling other Convex functions.**
   - Actions that call public `api.*` functions via `ctx.runQuery`/`ctx.runMutation` add unnecessary depth to the type tree.
   - Use `internalQuery`/`internalMutation` variants and call them via `internal.*` instead.

4. **Use static imports for `internal`.**
   - Always `import { internal } from "./_generated/api"` at the top of the file.
   - Never use `const { internal } = await import("./_generated/api")` inside function bodies — dynamic imports add type complexity.

### Key Files

- `convex.json` — codegen configuration (`legacyComponentApi: false`)
- `convex/authSetup.ts` — renamed from `auth.ts` to avoid conflict with `convex/auth/` directory
- `convex/_generated/api.d.ts` — regenerated by `npx convex codegen`

### Diagnosing TS2589

If TS2589 reappears:
1. Check for new file/directory naming conflicts in `convex/`.
2. Run `npx convex codegen` to regenerate types.
3. Run `npx tsc --noEmit` to verify the fix.
4. As a last resort for a specific call site, add an explicit `returns` validator to the function — this short-circuits type inference at that point.
